# Modernize Portfolio

Turn Kabeer Ali's multi-page portfolio into a single, continuous, scroll-driven page
that feels alive — components fade, slide, and lock into place as the user scrolls,
inspired by modern editorial agency sites (e.g. the Zohaib Aslam reference).

## Hard constraints (do not break these)

- **Stay on CRA.** Do NOT migrate to Vite/Next. The Netlify build depends on
  `NODE_OPTIONS=--openssl-legacy-provider` and Node 16 (see `netlify.toml`,
  `package.json` scripts). Any tooling change re-breaks the build.
- **⚠️ Tailwind is PRECOMPILED — regenerate `main.css` after every class change.**
  `src/index.js` imports `src/css/main.css`, a static file generated by the
  `build:css` script (`postcss src/css/tailwind.css -o src/css/main.css`). CRA 4
  **ignores `postcss.config.js`**, so Tailwind does NOT run live in `start`/`build` —
  any utility class that didn't already exist in `main.css` (e.g. a bare
  `bg-primary-dark`, `animate-marquee`, `backdrop-blur`, opacity modifiers like
  `bg-indigo-600/20`) will silently resolve to nothing. After adding/removing classes
  you MUST run, with the Node 24 OpenSSL flag set:
  `npx postcss src/css/tailwind.css -o src/css/main.css`
  (PowerShell: `$env:NODE_OPTIONS="--openssl-legacy-provider"; npx postcss ...`).
  New `keyframes`/`animation` go in `tailwind.config.js` first, then regenerate.
- **No new heavy dependencies.** `framer-motion@4.1.17` is already installed but is
  OLD: it has `motion`, `AnimatePresence`, `useViewportScroll`, `useTransform`,
  `useSpring`, `useAnimation` — but **NOT** `whileInView` or `useInView`. For
  reveal-on-scroll, use the project's own `useInView` hook
  (`src/hooks/useInView.jsx`, IntersectionObserver, zero deps) + `useAnimation` or a
  boolean `animate` prop. Don't reach for `whileInView`.
- **Keep the indigo/navy brand.** Use the existing Tailwind tokens — `primary-dark`
  `#0D2438`, `secondary-dark` `#102D44`, `ternary-dark` `#1E3851`, and `indigo-500`
  as the single accent. Dark mode (`darkMode: 'class'`, `useThemeSwitcher`) must keep
  working. Note: `useThemeSwitcher` defaults to `'dark'` and ignores any non
  `light`/`dark` value in `localStorage.theme` (it previously stored the string
  `"undefined"` and added a bogus `"undefined"` class to `<html>`, silently disabling
  dark mode on first load).
- **Reuse real content.** Pull from `src/data/*` (`aboutMeData`, `clientsData`,
  `projects.js`) and existing images in `src/images/`. Never invent fake projects,
  stats, or contact info.
- **Fonts:** GeneralSans variable. Use the existing utility classes
  `font-general-semibold`, `font-general-medium`, `font-general-regular`.

## Architecture target

One-pager lives on `/` (`src/pages/Home.jsx`). Sections stack vertically, each in
`src/components/sections/`. The other routes (`/projects`, `/about`, `/contact`) may
stay for deep-linking but the primary experience is the single scroll. Header nav
uses `react-scroll` (already a dependency) to smooth-scroll to section anchors.

Section order:
1. **Hero** — full-viewport, dark, oversized name + role, profile image, "scroll" cue.
2. **About / How I help** — bio with word-by-word scroll-reveal (greyed → solid as it
   enters view), numbered like `(01)`.
3. **Stack** — the 12 tech logos from `clientsData`, as an infinite marquee + reveal.
4. **Projects** — the 6 real projects as interactive cards (hover lift, image zoom,
   staggered entrance).
5. **Stats** — reuse `AboutCounter` / `CounterItem`.
6. **Contact** — reuse `ContactForm` + `ContactDetails`, restyled dark.
7. **Footer** — minimal, socials.

## Interaction toolkit (the "interactive feel")

- `useInView(ref, { once, margin })` → boolean. Drives entrance animations.
- `<Reveal>` (`src/components/reusable/Reveal.jsx`) — wraps children, fades + slides
  up on enter, supports `delay` for stagger. Default building block.
- **Scroll-linked** motion via `useViewportScroll()` + `useTransform()` for parallax,
  progress bars, and pinned headlines.
- **Stagger** lists by passing incremental `delay` (e.g. `i * 0.08`).
- Respect `prefers-reduced-motion` — gate large movement so it degrades to a fade.
- Keep animations fast (0.5–0.9s, `easeInOut`) and subtle; never block content from
  appearing if JS/observer fails (start visible, enhance).

## Workflow

1. Read current state: `Home.jsx`, `AppBanner.jsx`, `AppHeader.jsx`, the data files.
2. Build/verify infra first: `useInView` hook + `Reveal` component.
3. Build sections one at a time, newest at top of the file, verifying the page still
   compiles after each (`yarn start`).
4. Wire header nav to section ids via `react-scroll`.
5. Verify the production build: `yarn build` (with the openssl legacy flag) before
   declaring done — Netlify will run it.

## Verify

- `yarn start` → scroll the whole page; every section animates in once, smoothly.
- Toggle dark/light — indigo brand holds in both.
- Resize to mobile (≤640px) — layout reflows, text scales, no overflow.
- `yarn build` succeeds with no new warnings-as-errors.
